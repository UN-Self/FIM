## 我们押的注

今天的 AI 编程工具，默认都押着同一件事：你迟早要退场。它们抢着在你敲完字之前弹出建议，抢着把一整个文件砸到你脸上，然后管这叫生产力。它们提示、补全、自动合并，一步步把"你不再被需要"的未来变成现实。

FIM 押的是相反的注——你哪儿也不去。

它的工作不是替你想，是在你决定了墙往哪儿砌的那一瞬间，把下一块砖递过来。

## FIM 不是什么

不是聊天。不是 Agent。不是霸占你编辑器的侧边栏。不是一键生成一整个应用、然后拍拍屁股走人的"一锤子买卖"。

FIM 是，且只是：中间那一段——你写完注释之后、画完函数骨架之后，填进去的那段代码。两端由你定义，它填中间。名字就是这么来的：**Fill-in-the-Middle**。

## 三个承诺

### 沉默，直到被唤起

你进入心流的时候，FIM 是隐形的。不弹窗，不打扰，不会在你的下一个念头成型之前就用幽灵文本抢答。那些每次按键都弹出来的工具，不是在帮你思考——是在跟你抢下一个词。

想用 FIM 的时候，按 `Alt+\`，只在那个时候。它悄悄浮现一条建议，然后等着。一个工具尊不尊重你，看它懂不懂什么时候闭嘴。

### 你的意图，它的实现

注释是你和机器之间最诚实的契约。当你写下 `// 二分查找，返回索引，找不到返回 -1`，最重要的设计其实已经做完了，剩下的只是翻译。

FIM 守住这条。你写注释，或写函数签名，或写下第一个分支——然后 FIM 填充函数体。一次一个函数，绝不一次吐一整个文件。中间被填上了，两端始终是你的。

至于检索性知识——那个呼之欲出的库调用、那个你本要去查的参数顺序——那是 FIM 的活，不是你的。你只管说清意图（`deduplicate_and_sort(items)`），FIM 把它解析成你项目里最合适的那行调用。你的脑子始终留在"要什么"，而不是"哪个重载"。

### 你的机器，你的模型

无遥测。无云端分析。没有使用数据偷偷往外传。除非你自己把 FIM 指向某个 provider，否则任何东西都不离开你的机器。

自带模型。用 [Ollama](https://ollama.com) 跑一个本地的，或塞进任何 OpenAI-compatible 的 API——OpenAI、Anthropic、Mistral、Groq、Gemini、你自己的 vLLM。密钥、endpoint、那个关掉的开关，全在你手里。FIM 是宿主，不是租客。

## 补全，不是聊天

Chat 一次甩给你一整个代码库。你审不过来，也不敢动，只好继续叫它改——于是它越改越大，越改越烂，三个礼拜之后你在 debug 一坨没人看得懂的代码，最看不懂的就是你自己。

补全逼出一种不一样的节奏。一个函数。一个决定。一次一块砖。这才是你真正思考的方式——把大的拆成小的，一个一个解决，沿途每一个选择都是你自己做的。

FIM 押注补全，因为补全从设计上就把你留在环里。这里没有可以打瞌睡的自动驾驶。

## 指挥

你不需要一个替你把房子盖完的机器人。你需要一个在你砌墙时递砖的人。

FIM 是乐手。你才是指挥。

> 别再让它猜了。你来指挥。

## 试一下

FIM 是一个 VS Code 扩展。三步。

1. 从 [releases](https://github.com/UN-Self/FIM/releases) 装 `.vsix`：`code --install-extension fim-*.vsix`
2. 指向一个模型——本地 Ollama，或任何 OpenAI-compatible endpoint。
3. 写一条注释。画一段骨架。按 `Alt+\`。

它为什么这样设计——编排、FIM 模板、本地优先架构——都在 [`docs/`](./docs) 里：

- [`docs/PD.md`](./docs/PD.md)——产品与技术设计（那笔注的完整版）
- [`docs/providers.md`](./docs/providers.md)——支持的 provider
